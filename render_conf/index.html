Front end testing

Some frameworks you may have heard of:

  • Webdriver.io
  • Jasmine
  • Chai
  • Mocha
  • PhantomCSS
  • BackstopJS

Things to remember:

  • Focus on outcome, not input
  • Testing on as small pieces as possible, units
  • Acceptance test – making sure all small pieces work well together
  • Visual Regression testing (phantomCSS, BackstopJS)
    • Test components rather than full pages
    • CSS testing could focus only on style guide instead of the whole app if we could use it as a single source of truth

Accessibility testing!

  • Front end testing is time consuming so use effectively

FLOW-MATIC

First English-like data processing language developed under Grace Hopper, between 1955 and 1959.

FLOW-MATIC code sample

Programming languages, and software engineering have been around for a while.

The web is a newer aspect of it.

Many of the technical problems we have to deal with on the web as it grows have been problems in computer science for half a century. Let's use some of the well known concepts to our benefit.

DRY

Don't Repeat Yourself

  • Don’t repeat KNOWLEDGE / INFORMATION
  • It’s ok for the output to have duplicates, DRY is about maintenance
  • Relevant informations should have a single representation in the system

Repetition is better than complex abstraction

Use mixins, variables, no same change several times

Single source of truth

Single responsibility principle

Separation of concerns – each thing responsible for itself

Should be able to go from semantics, assistive technology markup, styles, to js functionality one-by-one, and change each aspect of the system without affecting others, i.e. don’t target things from one layer in another layer as it creates dependencies e.g. Do not target [data-attr] or .js-classes in CSS

Immutability

"State which cannot change"

Responsive suffixes class = ".class@md"

Conditional states - only be this thing in a certain view context

Open/Closed principle "open for extension, closed for modification"

  • Rather than mutating something by overriding, better to use modifier classes
  • Never mutate when overriding – always use classes.
  • Extend rather than modify
